# Passages and Navigation

## Passage

A **passage** is a single chunk of story content. Sugarbox is headless, so a passage’s `data` can be anything you want to render, like plain text, markdown, HTML, or even a component.

Passages are objects with:

- `name` (string): a unique id for the passage
- `data` (any): the passage content payload
- `tags` (optional string[]): any labels you want to attach for filtering or UI

You must provide a `startPassage` when you initialize the engine. You can also preload more passages via `otherPassages`, and add or overwrite passages later with `addPassage()` / `addPassages()`.

The current passage and its id are available via `engine.passage` and `engine.passageId`.

## Navigation

To move between passages, call `navigateTo(passageId: string)`.

When navigation happens, the engine fires a `:passageChange` event.

In this repo’s engine code, the recommended way to subscribe is the engine’s typed event API:

- `engine.on(eventName, listener)` (returns an unsubscribe function)
- `engine.off(eventName, listener)` (optional alternative)

Example:

```tsx
const off = engine.on(":passageChange", (event) => {
  const { oldPassage, newPassage } = event.detail;
  console.log("from", oldPassage?.name, "to", newPassage?.name);
});

// later
off();
```

Important behavior detail: passage navigation also triggers a `:stateChange` event (because passage id is stored in state metadata), so UIs that only listen to state may still re-render on navigation.
